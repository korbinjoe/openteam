为什么 Recovered 必须用琥珀色,而不是红/绿?
Agent 探索式工作中,工具重试是常态(找文件路径、调整参数、迭代 grep)。
▸ 涂红 = 喊"狼来了",用户每次进来都看到一片火 → 注意力被消耗;
▸ 完全绿 = 隐瞒过程波折,用户无法回溯"为什么这个任务跑了 5 分钟";
▸ 琥珀 = "成功了,但不是一次过" — 信息存在但不打扰,需要时点开看。
这与 CLAUDE.md 中"done 状态应该刻意 recessive"的原则一致 — 同理,recovered 状态也不应该抢注意力。

核心原则 · 借鉴 CLAUDE.md 里 "done 状态应该刻意 recessive,不该和 running 抢注意力" 的设计哲学:
① 注意力梯度 error > running > waiting > done > idle;
② 同一状态在色点 / 色条 / 文字色三处用同一颜色,绝不让用户因为"绿条配红字"困惑;
③ Done 状态删除一切重型视觉(色条/spinner/banner),只保留 muted 绿点。
