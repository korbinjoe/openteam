UX Proposal · OpenTeam

让左侧菜单更安静，更有用

当一个用户管理 100+ 个并行任务时，sidebar 是他每天看 200 次的地方。 现在它太"吵"——重复的标题、相同的徽章、反复出现的「Add Agent」、悬停才出现的小图标——一切都在抢注意力。

状态：已观察实测 · 8 处主要问题
主张：从「列表」升级为「指挥台」
Round · 1
TL;DR

三件最该改的事

  1. 01
    把每个任务从 ~80px 压到 36px。默认折叠 agent 行与 Add Agent 行，hover/active 才展开。
  2. 02
    加一条搜索 + 状态筛选条。218 个任务里 80% 是历史归档，但当前没有任何过滤手段。
  3. 03
    用真实状态点（运行 / 等待 / 完成 / 错误）替代统一的紫点。脉冲式工作流的核心是「我离开期间，谁在动？」
Why now

OpenTeam 的核心节奏是「批量派发 → 离开 → 批量验收」。
sidebar 是回到台面后第 1 秒看到的东西。 现在它要求用户用 5 秒滚动 + 阅读才能搞清「谁动了、谁卡了、谁要我审」。 目标：把这 5 秒缩到 1 秒。

A1 每条任务占 ~80px

标题 + agent + Add Agent 三行常驻。113 个任务占 9000+px 垂直空间，必须滚才能扫到一周前的。

A2 大量「Generate a concise title…」

title 自动生成失败时显示原始 prompt 模板，连续多个标题完全相同，列表退化为不可识别。

A3 Pin / Archive 仅 hover 出现

16×16 极小图标贴在已截断标题右侧，出现时还会进一步压缩标题宽度，触发 layout shift。

A4 紫色状态点没有语义

所有任务都是同一个紫点。无法一眼看出谁在跑、谁卡了、谁出错了——脉冲式工作流的核心信号缺失。

A5 活动高亮在 agent 行而非任务

当前选中态出现在子项 agent，而不是父项任务。点击 title vs agent 是两个不同跳转，用户难以建模。

A6 没有搜索 / 筛选 / 分组

218 个任务在列，找一个昨天的需要肉眼滚动 30 屏。没有 status filter，没有 today/yesterday 分组。

A7 折叠侧栏 = 失去全部任务

⌘B 折叠后只剩 logo + 4 个底部图标。当前任务、活跃任务全部消失，回不去。

A8 底部导航占 ~150px

Team / Skills / Schedules / Workspaces 四行 + 4 个图标 + theme 切换器，全部并列堆叠，永久占用任务列表的可视空间。

Section 02

设计原则

P1

注意力优先

默认安静。只有需要用户做决策的任务才"喊"——状态变化、需要审、被阻塞。

P2

脉冲友好

回到台面 1 秒内看到："正在跑 3 个 / 等审 2 个 / 失败 1 个"。无需滚动。

P3

渐进披露

默认每行 36px 极简。hover 显行动，active 展开 agents，永远不主动占空间。

P4

批量原生

超级个体每天派 20 任务、归档 50 任务。多选、批量归档、批量重派必须是一等公民。

Section 04

交互细则

状态点语义

点含义可达动作
Running · 有 agent 正在产出 token点击 → 跳到对话尾部 / 监听 stream
Awaiting · 等待用户输入或确认点击 → 直接定位到 prompt 区
Idle / Completed · 静默悬停可 Pin / Archive
Error · session 失败 / 超时点击 → 显示错误 + Re-run

快捷键

新建任务⌘N
搜索 / 跳转⌘K
折叠 sidebar⌘B
下一任务J
上一任务K
归档当前E
Pin 当前P
多选模式X

行的 5 种态

Default · 静态36px
Hover · 显 actionsopacity tween · no shift
Active · 展开 agents~76px
Selected · 多选模式checkbox
Error · 红边提示需用户处理

分组与排序

默认按 状态优先 → 时间次之：

  1. Error（最显眼，要求处理）
  2. Awaiting（卡在等用户）
  3. Running（正在跑）
  4. Today · Yesterday · This week · Older（按完成时间）

用户可切到 仅时间 或 仅按 agent 模式。Pinned 永远置顶。

实现路径建议

Phase 1 · 减噪

  • 任务行默认折叠（隐藏 Add Agent + agent 行）
  • 状态点改用 4 色语义（要 server 先上报状态）
  • actions 用 opacity 切换避免 layout shift

Phase 2 · 加力

  • 顶部搜索 + 状态 chips（client 端过滤）
  • Today / Yesterday 时间分组
  • 修 title 生成失败回退（避免连续多个 "Generate a concise title…"）

Phase 3 · 进阶

  • 多选 + 批量操作浮条
  • 折叠态命令台 popover（hover Logo 触发）
  • 键盘导航 J/K/E/P/X
