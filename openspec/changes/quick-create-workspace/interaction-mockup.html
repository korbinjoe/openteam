新旧流程对比

当前流程 (4 步 + 多次点击)

1 点击 "+ New" 按钮
2 输入 Workspace 名称
3 点 "添加 Repo" → 弹出目录选择器 → 导航 → 选择
4 回到创建 Dialog → 点 "创建并开始"
中间有 2 个 Dialog 串联，认知负担重

新流程 (1 步操作 + 1 步选择)

1 点击 "+ New" → 直接弹出文件夹选择
2 导航/搜索 → 选中文件夹 → 完成
系统自动完成：
• Workspace 名称 = 文件夹名
• 自动绑定为单 Repo
• 创建完成后自动跳转进入
• 已存在时直接跳转（不重复创建）
边界场景处理
文件夹已被绑定 该路径已属于某 Workspace → 直接跳转进入已有 Workspace，toast 提示 "该项目已在 Workspace「xxx」中"
名称冲突 已有同名 Workspace → 自动追加后缀：ai-saas-app → ai-saas-app-2
多 Repo 需求 先用一键创建（单 Repo），进入后在 Workspace 设置中追加更多 Repo。覆盖 80% 单项目场景
目录导航 Electron：原生 OS 对话框处理（Finder/Explorer 多级导航、搜索、收藏夹）
Web：DirPickerDialog 逐层浏览 + 全局搜索直达
取消 / ESC 关闭选择器，无任何副作用，回到 Workspace 列表
空文件夹 允许选择空文件夹创建 Workspace（用户可能要新建项目）
实现要点
前端改动 WorkspacesPage.tsx：
• 点击 New → 判断 isElectron
• Electron → 通过 IPC 调用 dialog.showOpenDialog
• Web → 直接打开 DirPickerDialog（跳过 CreateWorkspaceDialog）
后端复用 复用已有 POST /api/workspaces/quick-start 接口：
• 传入 repoPath → 自动 find-or-create workspace
• 名称取 basename(repoPath)
删除代码 CreateWorkspaceDialog.tsx 可删除（或保留为 "高级创建" 入口）
Electron IPC 新增 IPC handler：ipcMain.handle('pick-directory', () => dialog.showOpenDialog(...))
